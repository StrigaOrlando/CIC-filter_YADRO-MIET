# MATLAB golden model CIC-фильтра

В архиве находится первая версия MATLAB golden model для CIC-дециматора 

Модель предназначена для генерации входных воздействий и эталонных выходных данных, с которыми затем сравнивается RTL-реализация

## Что есть в модели:

- CIC-дециматор;
- signed integer в допкоде;
- wrap-around при переполнении;
- параметры IN_WIDTH, OUT_WIDTH, R, M, N;
- полная и неполная выходная разрядность;
- режимы округления:
  - trunc — усечение;
  - ceil — округление в +inf;
  - fix — округление к нулю;
- нормировка для случая, когда коэффициент усиления является степенью двойки;
- генерация файлов для RTL testbench;
- сравнение выхода RTL с эталоном MATLAB

## Основной сценарий работы

### Шаг 1. Сгенерировать вход и эталонный выход

Открыть MATLAB в папке с распакованным архивом и запустить:

```matlab
run_generate_vectors
```

После запуска появится папка:

```text
rtl_test_case/
```

В ней будут файлы:

- input.txt
- expected_output.txt
- after_integrators.txt
- after_decimator.txt
- after_combs.txt
- info.txt

Для RTL в первую очередь нужны:

- input.txt – подаём на вход RTL testbench
- expected_output.txt – эталонный выход MATLAB golden model для сравнения

### Шаг 2. Прогнать RTL

В RTL testbench нужно считать входные данные из файла:

```text
rtl_test_case/input.txt
```

После работы RTL нужно сохранить выходные данные в файл:

```text
rtl_test_case/rtl_output.txt
```

Формат файла: одно целое число в одной строке

### Шаг 3. Сравнить RTL с MATLAB golden model

После появления файла rtl_test_case/rtl_output.txt нужно снова открыть MATLAB и запустить:

```matlab
run_compare_rtl_output
```

Скрипт сравнит:

```text
rtl_test_case/expected_output.txt
rtl_test_case/rtl_output.txt
```

Если всё совпало, будет сообщение:

```text
OK: RTL output полностью совпадает с MATLAB golden model.
```

Если есть ошибки, MATLAB выведет:

- количество несовпадающих отсчётов
- первую позицию ошибки
- ожидаемое значение
- значение RTL
- максимальную абсолютную ошибку
- RMS-ошибку (среднюю ошибку)

## Настройка параметров теста

Параметры задаются в файле:

```text
run_generate_vectors.m
```

Основная строка конфигурации:

```matlab
cfg = make_cfg(16, 16, 4, 1, 2, "full", "trunc", false);
```

Порядок параметров:

```text
make_cfg(IN_WIDTH, OUT_WIDTH, R, M, N, OUTPUT_MODE, ROUND_MODE, NORMALIZE)
```

Пример:

```matlab
cfg = make_cfg(16, 16, 4, 1, 2, "full", "trunc", false);
```

означает:

```text
IN_WIDTH = 16
OUT_WIDTH = 16
R = 4
M = 1
N = 2
OUTPUT_MODE = full
ROUND_MODE = trunc
NORMALIZE = false
```

## Выбор входного воздействия

В файле run_generate_vectors.m можно выбрать тип входного сигнала.

По умолчанию включён постоянный вход:

```matlab
x = 100 * ones(num_samples, 1);
```

Чтобы выбрать другой сигнал, нужно закомментировать текущий вариант и раскомментировать нужный:

- одиночный импульс;
- ступенька;
- синус в полосе пропускания;
- синус около первой зоны подавления;
- псевдослучайный signed-сигнал;
- максимальные положительные и отрицательные значения.

## Фаза децимации

В golden model по умолчанию используется:

```text
DECIMATION_PHASE = R
```

Это означает, что первый выходной отсчёт после децимации формируется на R-м входном отсчёте.

RTL testbench должен использовать такую же фазу, иначе выход может быть сдвинут относительно MATLAB-эталона.

## Графики

Для визуальной отладки можно использовать:

```matlab
run_cic_plot_example
```

В этом скрипте можно включить или выключить графики:

```matlab
cfg.PLOT_ENABLE = true;       % включить графики
cfg.PLOT_INTERNAL = true;     % показать внутренние сигналы
cfg.PLOT_MAX_SAMPLES = 100;   % сколько отсчётов показывать
```

Для массовой генерации тестов графики лучше держать выключенными
